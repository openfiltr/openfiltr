# Governance

## Project vision

OpenFiltr aims to be the most developer-friendly, community-governed DNS filtering platform for self-hosted networks.

## Decision making

We use **lazy consensus**: a proposal is accepted if no objections are raised within 7 days of posting. Significant changes go through the RFC process.

## RFC process

1. Open a GitHub issue using the RFC template.
2. 14-day comment period.
3. Maintainers close with accepted/rejected status.

## Maintainer responsibilities

- Review and merge pull requests within a reasonable time.
- Participate in RFC discussions.
- Cut releases and maintain the changelog.

## Conflict resolution

Unresolved conflicts are escalated to a maintainer vote. Simple majority carries.
